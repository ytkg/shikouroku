# 関連嗜好機能仕様

- 更新日: 2026-03-05
- 対象: `apps/web` / `apps/api`
- 種別: 機能仕様
- バージョン: v1.1（実装済み）

## 1. 機能概要

- ある嗜好（entity）に別の嗜好を関連付ける。
- 関連は無向関係として扱う（A-B を 1 レコードで管理）。

## 2. 画面仕様

### 2.1 詳細画面

- 関連が 1 件以上ある場合のみ `関連嗜好` セクションを表示。
- 各関連嗜好はトップページの嗜好カードと同等の見た目・情報量で表示する。
- 表示要素は `名前` / `種別(または気になる+種別)` / `説明(1行省略)` / `タグ` / `画像サムネイル`。
- 各関連嗜好カードはクリックで詳細へ遷移する。
- 遷移時は現在の一覧クエリ（`location.search`）を引き継ぐ。

### 2.2 新規登録画面

- `関連を編集` ダイアログで候補を複数選択できる。
- ダイアログ内で `候補名の部分一致検索` と `種別絞り込み` を提供する。
- ダイアログに選択中件数（`選択中: n件`）を表示し、選択行を強調表示する。
- entity 作成後に選択済み ID へ順次 `POST /related` を実行する。

### 2.3 編集画面

- `関連を編集` ダイアログで現在の関連を編集。
- 関連嗜好の表示カードはトップページの嗜好カードと同等の見た目・情報量で表示する。
- 編集画面の関連嗜好カードはクリック遷移しない（`関連を編集` ダイアログ導線を維持）。
- 候補一覧は `GET /api/entities` 結果をベースにしつつ、現在関連済みだが一覧から漏れる項目もマージして表示する。
- 自分自身（編集中 entity）は候補から除外する。
- 保存時に差分計算し、追加分は `POST`、削除分は `DELETE` を順次実行する。
- `409 (already exists)` と `404 (already removed)` は冪等扱いで継続。

## 3. API 仕様

- `GET /api/entities/:id/related`
  - `200`: `{ ok: true, related: Entity[] }`
- `POST /api/entities/:id/related`
  - body: `{ relatedEntityId: string }`
  - `201`: `{ ok: true }`
- `DELETE /api/entities/:id/related/:relatedEntityId`
  - `200`: `{ ok: true }`

### 3.1 エラー

- `400`: 自己関連（`id === relatedEntityId`）
- `404`: entity 不在 / relation 不在
- `409`: 既存 relation の重複作成
- `401`: 未認証
- `500`: relation 永続化処理失敗

## 4. データモデル

`entity_relations`

- 主キー: `(entity_id_low, entity_id_high)`
- 制約: `entity_id_low <> entity_id_high`
- 並び: `created_at DESC` で関連一覧を返却
- 保存時は ID の辞書順で正規化して重複を防止

## 5. 業務ルール

1. 自己関連は禁止。
2. 重複関連は禁止。
3. どちらかの entity が存在しない場合は作成/削除しない。
4. 片側で作成した関連は反対側詳細でも表示される。
5. 編集時の関連同期は差分適用（追加/削除）で行い、競合済み/削除済みは冪等として継続する。

## 6. テスト観点（現行）

- API application（relation create/delete/list）の正常・異常系
- API parser/client の正常・異常系（関連レスポンス正規化、関連APIパス、不正レスポンス）
- create/edit での関連選択ロジック（追加・削除・差分計算）
- 関連編集ダイアログの候補絞り込み（候補名検索、種別フィルタ、空状態、選択件数表示）
- API パス生成と URL エンコード

## 9. 既知の制約

- 関連候補は `GET /api/entities` を全ページ取得してクライアント側で絞り込むため、件数増加時は初回取得コストが上がる。
- 候補の検索は候補名の部分一致のみで、説明文・タグ横断の候補検索は未対応。
- 候補の並び替えは専用要件未定義のため、一覧取得順（作成日時降順）に依存する。
